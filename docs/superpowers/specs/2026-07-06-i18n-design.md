# i18n Design

Route: openspec+superpowers

## Goal

Add internationalization support for RhythHaus shared Compose UI strings, with English as the default language and Simplified Chinese resources as the first translated locale.

## Design

Use Compose Multiplatform resources, which are already enabled in the `shared` module. User-facing shared Compose copy moves from hardcoded string literals to `stringResource(Res.string.*)` calls. Dynamic text continues to compose runtime values such as track names, artist names, counts, and search queries, but the surrounding labels and formats come from resources.

## Scope

In scope:
- Shared Compose UI screens/components: Library/Home, NowPlayingBar, NowPlayingScreen, Search, Settings, BackChip, dialogs, scanner card, content descriptions.
- Default `values/strings.xml` and Chinese `values-zh/strings.xml` resources under `shared/src/commonMain/composeResources`.

Out of scope:
- User media metadata (track/artist/album values).
- Platform-native app names beyond existing Android `app_name`.
- Domain/backend diagnostic strings that are not presented as shared UI copy.

## Acceptance Criteria

- Shared Compose UI builds using Compose resource strings.
- English default resources and Chinese resources exist.
- Existing playback, navigation, scanner, Haze chrome, and roadmap behavior remain unchanged.
- `./gradlew :shared:compileKotlinJvm :shared:jvmTest --configuration-cache` passes.
