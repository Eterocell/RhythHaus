# i18n Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Complete Compose Multiplatform resource-backed English and Chinese UI strings for shared Compose screens by adding missing keys, removing dead keys, and migrating the remaining hardcoded shared UI copy.

**Architecture:** Extend `shared/src/commonMain/composeResources` with the missing keys and remove unused keys. Replace the remaining hardcoded strings in `LibraryAppShell`, `LibraryRows`, `ImportLabels`, `App`, and the platform folder-picker launchers with `stringResource` calls.

**Tech Stack:** Kotlin Multiplatform, Compose Multiplatform resources, existing shared Compose UI.

## Global Constraints

- Do not alter navigation, playback, scanner, Haze chrome, persistence, or platform entry-point behavior.
- Do not translate user media metadata (track/artist/album values) or domain log/diagnostic strings.
- Keep English as default and add Simplified Chinese resources.
- Use existing Compose Multiplatform resources; do not add a new i18n dependency.
- Prefer deleting dead resource keys over leaving them untranslated.
- Preserve the existing package structure; `ImportLabels` may be collapsed to commonMain composable functions.
